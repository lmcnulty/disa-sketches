I was told that we don't want to have owners as persons in the database. I was wondering whether it might be preferable to include them in the database but hide them from the public-facing interface.

It seems plausible that someone might want to look at all the slaves owned by the same person. If we just enter owners as a name attached to each slave, then we're likely to have problems if we have different owners with the same name. Also, searching for matching strings is probably going to be slower than searching for matching IDs.

Additionally, we could encounter instances where an owner was also a family member, or maybe even cases where a slave was freed and then became an owner.

In order to handle these problems, I suspect that we're either going to need to include owners as persons, or we're going to need a separate type of entry for owners which would end up being used a lot like a person entry anyway, as it would have at least some of the same fields "eg: name" and it would be capable of having relations with persons.
